# カスタマイズ

カスタマイズは、winsmux をローカルの運用に合わせるためのものです。共有資格情報の仲介役にするものではありません。

## ワークスペース方針

`winsmux init` の既定値は、ワーカーごとに管理されたワークツリーです。

```powershell
winsmux init --workspace-lifecycle managed-worktree
```

複数のエージェントが並列で編集する場合は、管理ワークツリーを使ってください。オペレーターが採用するまで、変更を分けて扱えます。

## 起動プリセット

比較向けの実行を始める前に、プリセットを確認できます。

```powershell
winsmux launcher presets --json
winsmux launcher lifecycle --json
```

プリセットはワークスペースの形を表します。任意のセットアップや後片付けスクリプトを実行する仕組みではありません。

## エージェントスロット

winsmux は固定されたベンダー役割ではなく、スロットと能力で扱います。スロットは、作業、レビュー、相談などの能力を示せます。どのペインに何を任せるかはオペレーターが決めます。

割り当て時は次の考え方を使います。

- 実装は作業可能なスロットに送る
- レビューはレビュー可能なスロットに送る
- 難しい、曖昧、収束しない作業だけ相談を使う

## 資格情報

ペインに差し込む必要がある資格情報は Windows DPAPI で保存します。

```powershell
winsmux vault set <name> <value>
winsmux vault inject <name> <pane>
```

winsmux は OAuth flow を代理実行しません。callback URL を受け取らず、同じ PC 上での対話ログインの token を別ペインへ共有しません。

## デスクトップ設定

デスクトップアプリは CLI と同じオペレーター契約に従います。テーマ、表示密度、折り返し、コード用フォント、集中モードなどの設定を表示できますが、生の PTY 出力は診断用の端末ドロワーに残します。

デスクトップアプリは次の用途に使います。

- 実行証跡と判断 gate を確認する
- 記録済みの作業を比較する
- ソース単位の文脈へ掘り下げる
- 端末診断を残しつつ、主画面を証跡中心に保つ
