# ソースアクセスと再配布

このページは、`v1.0.0` 系から適用するソース公開と再配布の境界を説明します。

## 公開配布

`v1.0.0` 以降の winsmux は、完全な実装ソース一式ではなく、リリース済みの製品として公開配布します。公開リリース面に含めるものは次の通りです。

- Windows デスクトップインストーラー
- MSI 配布物
- npm パッケージと CLI インストール経路
- チェックサムファイル
- リリースノート
- 利用者向けとコントリビューター向けの公開ドキュメント
- バグ、サポート、機能要望のための公開 issue

公開リポジトリには、ドキュメント、リリースメタデータ、例、セキュリティ方針、互換性メモを残せます。ただし、すべての実装ソースを公開する場所ではなくなります。

## 有料会員向けソースコミュニティ

一部の実装ソースは、レビュー、学習、改善提案のために Substack 有料会員へ共有できます。このアクセスは会員向けコミュニティプログラムであり、オープンな再配布許可ではありません。

会員向けソースプログラムは、次のルールに従います。

- 共有するソーススナップショットはメンテナーが選びます。
- 秘密情報、ローカルパス、資格情報、非公開プロンプト、リリース鍵は共有前に取り除きます。
- 再配布、ミラー作成、再公開は許可しません。
- アクセス経路は、非公開リポジトリ、会員限定ダウンロード、またはメンテナーが選ぶ会員限定チャネルにします。
- 会員資格またはレビュー企画が終了した場合、アクセスを取り消せます。
- 貢献は提案またはパッチとして扱い、公開リリースへ入れる前にメンテナーが受け入れ判断をします。
- メンテナーが公開済みのソースを除き、公開 issue や公開ドキュメントで会員限定ソースを引用してはいけません。

## 境界

公開利用者は、ソースアクセスなしでインストール、更新、検証、issue 報告ができます。有料会員向けソースアクセスは、レビューとコミュニティ貢献のためのものです。winsmux を再配布したり、競合する公開ソースミラーを作ったりする権利は含みません。

セキュリティ報告には公開のセキュリティ方針を使ってください。秘密情報や顧客の非公開データを会員向けの議論チャネルに送らないでください。
