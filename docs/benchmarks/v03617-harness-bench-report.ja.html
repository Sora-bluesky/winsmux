winsmux Harness Bench Report

v0.36.17 では、ワーカーの印象比較ではなく、Harness Bench 方式で winsmux の実 worker pane を測定する。Kimi K2.7 Code と GLM-5.2 は候補表示ではなく、OpenRouter 経由の api_llm worker として扱う。

対象版: v0.36.17 測定方式: Harness Bench 正式結果: OpenRouter API key 待ち 公開条件: GitHub Release 公開まで

結論

このレポートは測定準備とブロック証跡の版です。正式結果は、27件の課題、hidden tests、sanitized workspace、60分 timeout、deterministic scoring を満たす実測が完了するまで空欄にします。

既存の9問 pack は smoke や下準備には使えますが、v0.36.17 の公開スコアには使いません。

OPENROUTER_API_KEY が未設定のため、OpenRouter Kimi K2.7 Code と GLM-5.2 は通信前に api_llm_api_key_env_missing で停止しました。これはモデル性能ではなく、除外理由付きの環境ブロックです。

測定手法

採点

一次判定は hidden tests と deterministic checks。LLMレビューは失敗監査と品質補助に限定し、自己申告スコアは使わない。

環境

各 worker に同じ task set、同じ timeout、同じ workspace 条件を適用する。operator は採点対象外。

除外

missing key、timeout、crash、empty stdout、invalid output は、スコアから除外し、理由を別表で記録する。

秘匿

API key、raw prompt、local user name、provider request id、生ログはHTMLへ出さない。

失敗分析

正式測定後、失敗を「テスト不通過」「timeout」「起動失敗」「認証不足」「出力形式不正」「operator介入過多」に分けて記録する。現時点の OpenRouter 行は「認証不足」であり、モデル品質スコアには入れない。

この章は、ランキングを補強するためではなく、winsmux の worker assignment と運用導線を改善するために使う。
